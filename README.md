## Day 36 ‚Äì ML Model 2: Logistic Regression on Titanic
- **Logistic Regression ‚Äî Real Dataset** (`ML Models/Logistic Regression/LogisticRegression_Titanic.ipynb`) ‚Äî End-to-end binary classification on the Titanic dataset predicting passenger survival. EDA: survival rates by sex, class, embarkation port, age, fare, and family size ‚Äî confirming the "women and children first" pattern quantitatively. Results: 82.7% accuracy, ROC-AUC 0.86, +21% over naive baseline. Confusion matrix and ROC curve. Coefficient interpretation ‚Äî each coefficient read as an odds multiplier (e.g. being female multiplies survival odds by 3.4√ó, each class step up multiplies odds by 0.39√ó). Threshold tuning ‚Äî precision/recall/F1 across all thresholds. Stratified 5-fold CV. Effect of regularization parameter `C` (inverse of alpha). Individual passenger prediction function.

Covers the maximum margin concept, support vectors, soft margin and C parameter, why StandardScaler is mandatory (unscaled: 93% vs scaled: 98.3%), kernel comparison (Linear, RBF, Polynomial, Sigmoid), C and gamma tuning via GridSearchCV (best ROC-AUC 0.9977), permutation importance as a substitute for built-in feature importances, and a full comparison against Logistic Regression and Decision Tree.

## Day 40 ‚Äì ML Model 6: K-Nearest Neighbors on Iris
- **KNN** (`ML Models/KNN/KNN_Iris.ipynb`) ‚Äî Classification on the Iris dataset (150 samples, 3 classes). No training phase ‚Äî stores all data and votes at prediction time. Covers distance metrics (Euclidean, Manhattan, Chebyshev), mandatory feature scaling, finding optimal K via CV, decision boundary visualization at K=1/5/15/30, GridSearchCV tuning, and Leave-One-Out CV.
